# SPEC.md — 愛知政治マップ Phase 1 仕様書

このドキュメントは Phase 1 の実装範囲を自己完結的に定義する。
全体方針・データソース・絶対ルールは `CLAUDE.md` を参照。

---

## 1. 目的と対象ユーザー

- **目的**：愛知県の有権者が「自分の地域の代表者が誰で、何を言い、どう投票したか」を
  国・県・市の三層でまとめて確認できるようにする。
- **対象ユーザー**：政治に詳しくない一般有権者。専門用語を避け、5分で全体像をつかめる導線。

---

## 2. Phase 1 スコープ

### 含む
- 議員一覧・議員詳細（国会＝愛知選出／愛知県議会／名古屋市会）
- 郵便番号・地図からの地域ナビゲーション
- 発言（一次ソースURL付き）とそのAI要約
- 採決（記名投票の賛否。記名でない採決はその旨を明示）
- 政治資金（出典リンク＋主要項目のみ）
- ローカル争点テーマ別ビュー
- 三層横串ビュー（同一争点で国・県・市の発言/動きを並置）

### 含まない（明確に対象外）
- 名古屋市以外の市町村
- 口コミ投稿、応援金/決済、ユーザー登録・ログイン
- リアルタイム更新（データは手動バッチで更新する前提）
- ネイティブアプリ、多言語対応

---

## 3. 対象議員の定義

- 国会：衆院 愛知1〜16区の議員、比例東海ブロックのうち愛知に関係する議員、参院 愛知県選挙区の議員
- 県：愛知県議会の現職議員
- 市：名古屋市会の現職議員
- 各議員に `level`（national / prefectural / municipal）を必ず付与する

## 4. データモデル（`/lib/types.ts` に定義）

```ts
type Level = "national" | "prefectural" | "municipal";

interface Legislator {
  id: string;
  name: string;
  kana: string;
  level: Level;
  party?: string;          // 会派・所属政党
  district: string;        // 例: "愛知1区" "比例東海" "名古屋市中区"
  photoUrl?: string;
  sourceUrl: string;       // 公式プロフィール等の一次ソース
}

interface Speech {
  legislatorId: string;
  date: string;            // ISO
  body: string;            // 会議名・議会名
  text: string;            // 発言全文または抜粋
  sourceUrl: string;       // 会議録の一次ソース（必須）
  aiSummary?: string;      // 中立・多視点の要約。元発言リンクを併記
}

interface Vote {
  legislatorId: string;
  billTitle: string;
  date: string;
  result: "yea" | "nay" | "absent" | "not_recorded"; // 記名でなければ not_recorded
  sourceUrl: string;
}

interface Funding {
  legislatorId: string;
  year: number;
  totalIncome?: number;
  totalExpense?: number;
  sourceUrl: string;       // 総務省 収支報告書へのリンク（必須）
}

interface Issue {           // 三層横串で扱うローカル争点
  id: string;               // 例: "linear" "ev-shift"
  title: string;            // 例: "リニア中央新幹線 名古屋開業"
  description: string;
  relatedSpeechIds: string[];
}
```

## 5. ページ構成（`/app` ルーティング）

- `/` … トップ。郵便番号入力＋愛知県地図。「政治の基本を5分で」導線
- `/area/[zip]` … 郵便番号→該当する小選挙区・県議選挙区・名古屋市の区と、その代表者
- `/legislators` … 議員一覧（level・会派・地域で絞り込み）
- `/legislators/[id]` … 議員詳細（プロフィール／発言＋AI要約／採決／政治資金。すべて出典付き）
- `/issues` … ローカル争点テーマ一覧
- `/issues/[id]` … 争点詳細＝**三層横串ビュー**（国・県・市の発言/動きを並置）
- `/about` … 中立性・データの出典・免責（公選法配慮の説明を含む）

## 6. ローカル争点テーマ（初期セット）

リニア中央新幹線 名古屋開業／自動車産業のEV転換と雇用／中部国際空港（セントレア）／
名古屋城天守の木造復元／製造業の外国人労働者／2026アジア・アジアパラ競技大会。
（各テーマは国政・県政・市政の論点を横串で結ぶことを優先する）

## 8. 非機能要件

- 静的生成中心で高速・低コスト。SEO（地域名・議員名で検索流入）を意識した構造化
- アクセシビリティ：公的議会サイトに準じ、JIS X 8341-3 のレベルA以上を目標
- AI要約コストはバッチで一括処理し、生成結果は `/data` にキャッシュして再生成を避ける

---

## 9. 中立性・法務ルール（再掲・必須遵守）

特定政党/候補者への投票・不投票を呼びかけない。すべてのデータに一次ソースURL。
選挙期間中の表示は投票誘導と取られない設計。robots.txt と各サイト規約を遵守。
不確実な法務判断は実装前に確認する。

---

## 10. 受け入れ確認（このフェーズの完了条件）

エンドツーエンドで以下が成立すれば Phase 1 完了とみなす:

1. トップで名古屋市内の郵便番号を入力すると、`/area/[zip]` に
   該当する衆院小選挙区・参院愛知・愛知県議選挙区・名古屋市の区と、その代表者が表示される。
2. 任意の国会議員（愛知選出）の詳細ページに、**一次ソースURL付きの発言が3件以上**と、
   その中立なAI要約（元発言リンク併記）が表示される。
3. 記名投票でない採決が「個人の賛否は非公開」と明示される。
4. 争点ページ（例：リニア）で、国・県・市それぞれの発言/動きが**1画面で並置**される。
5. `npm run build` `npm run typecheck` `npm run lint` がすべて成功する。
